🏗️ Recent Dashboard Architectural Upgrades

The system was recently refactored to support complex random data sets and provide deeper analytics:

  • Dynamic Schema Normalizer: Added frontend parsing to automatically normalize misnamed columns (e.g. Attendence(0-100) -> Attendance (%)) preventing formula breakage on randomized datasets.
  • Robust At-Risk Engine: Moved intervention logic from the UI into the core feature engineering pipeline. Students are now evaluated on three pillars: Avg Test Score (< 50%), Unit Test 3 (< 40%), and Engagement (< 40%).
  • Strict Subject Aggregation: Comprehensive Academic Profiles now strictly match students by Name + Class + Section to prevent data bleeding across different students with identical first names. Averages are mathematically computed across all matched subjects.
  • Radar Geometry Fallbacks: Patched Plotly string multiplication scaling bugs and implemented geometric fallbacks (auto-rendering Bar Charts when a student is filtered to fewer than 3 subjects, since polygons require 3+ points).
  • GenAI Action Plans: Integrated NVIDIA's meta/llama-3.3-70b-instruct model directly into the Deep-Dive view to generate formatted markdown action plans for Parents and Teachers based on the student's live analytics.
